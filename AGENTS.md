## Reglas No Negociables

- No se permiten respuestas genericas de ningun tipo
- Revisar el stack del proyecto (composer.json, package.json, etc)
- El agente debe usar la menor cantidad de tokens posible
- Las respuestas deben ser ajustadas al contexto del proyecto,
  - Si el proyecto es muy extenso en contexto, examinar el stack y luego el modulo que se este trabajando y como seria su integracion al proyecto en general

## Commits
    - el mensaje de commit debe ser semantico, es decir, indicar si es:
        - Feature
        - fixture
        - Delete
        - Refactor
        - etc
    - Debe tener un pequeño valor que aporta al proyecto (maximo 20 palabras)
    - Indicar los ficheros afectados y acciones sobre los mismos
